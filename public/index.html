Available Tools

Powerful Railway MCP tools to manage your Payload CMS projects with natural language.

Project Management

  • project_list - List all your Railway projects
  • project_info - Get detailed information about a project
  • project_create - Create a new Railway project
  • project_delete - Delete a project and all its resources

Database Tools

  • database_list_types - List available database types
  • database_deploy - Deploy a pre-configured database
  • volume_create - Create persistent storage for databases
  • volume_list - List all volumes in a project

Service Management

  • service_list - List all services in a project
  • service_create_from_repo - Deploy from GitHub
  • service_update - Update service configuration
  • service_restart - Restart a service

Deployment & Variables

  • deployment_trigger - Trigger a new deployment
  • deployment_logs - View deployment logs
  • variable_set - Set environment variables
  • domain_create - Create custom domains

Getting Started

Follow these steps to set up and start using the Payload CMS MCP Server.

1

Prerequisites

Before you begin, make sure you have:

  • Node.js 18+ (for built-in fetch API support)
  • An active Railway account
  • A Railway API token (create one at railway.app/account/tokens)
2

Configure Cursor

To use with Cursor IDE:

  1. Open Cursor Settings
  2. Go to MCP Servers section
  3. Add a new MCP server
  4. Name it "Railway MCP"
  5. Set Transport Type to "Command"
  6. Set Command to: YOUR_RAILWAY_API_TOKEN
  7. Save the configuration
3

Configure Claude for Desktop

To use with Claude for Desktop:

  1. Edit your Claude for Desktop config file:
    • macOS: ~/Library/Application\ Support/Claude/claude_desktop_config.json
    • Windows: %APPDATA%\Claude\claude_desktop_config.json
  2. Add this configuration: "mcpServers": { "railway": { "command": "YOUR_RAILWAY_API_TOKEN" } }
  3. Restart Claude for Desktop
4

Start Using

Once configured, you can use natural language to manage your Payload CMS projects:

  • "Create a new Payload CMS project with MongoDB"
  • "Set up a blog collection with title, content, and author fields"
  • "Configure authentication with email and password"
  • "Deploy my Payload CMS project to Railway"
